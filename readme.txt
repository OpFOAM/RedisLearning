客户端暂停 超时

起始版本：50年2月9日
时间复杂度： O（1）
CLIENT PAUSE 是一个连接控制命令，能够在指定的时间（毫秒）内挂起所有的Redis客户端。
该命令执行以下操作：
它停止处理来自普通客户端和发布/订阅客户端的所有未决命令。然而，与奴隶的互动将继续正常。
但是，对调用方ASAP返回OK，所以CLIENT PAUSE命令执行不会自行暂停。
当指定的时间已过时，所有的客户端都被解除阻塞：这将触发在暂停期间处理每个客户端的查询缓冲区中累积的所有命令。
这个命令是有用的，因为它能够以受控的方式将客户端从Redis实例切换到另一个实例。例如，在实例升级期间，系统管理员可以执行以下操作：
暂停使用的客户端 CLIENT PAUSE
等待几秒钟，以确保从服务器处理来自主服务器的最新复制流。
把其中一个奴隶变成主人。
重新配置客户端以连接新的主服务器。
可以发送CLIENT PAUSE一个MULTI / EXEC块和INFO replication命令，以便在客户端被阻塞时获得当前的主站偏移量。这样就可以等待从端的特定偏移量，以确保所有的复制流都被处理。

@ simple-string-reply：如果超时无效，则该命令返回OK或错误。

客户端SETNAME  连接名称

起始版本：2.6.9
时间复杂度： O（1）
该CLIENT SETNAME命令为当前连接分配一个名称。
分配的名称显示在输出中，CLIENT LIST以便可以识别执行给定连接的客户端。
例如，当使用Redis来实现队列时，消息的生产者和消费者可能希望根据其角色设置连接的名称。
如果不是Redis字符串类型（512 MB）的通常限制，则可以分配的名称长度没有限制。但是，在连接名称中不能使用空格，因为这会违反CLIENT LIST回复的格式。
可以完全删除连接名称，将其设置为空字符串，这不是有效的连接名称，因为它用于此特定目的。
连接名称可以使用检查CLIENT GETNAME。
每个新的连接都没有分配的名称。
提示：将名称设置为连接是使用Redis应用程序中的错误调试连接泄漏的好方法。
@返回
@ simple-string-reply：OK如果连接名称被成功设置。
